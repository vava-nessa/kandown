# Context

/

The File System Access API (FSAA) is only available in Chromium-based browsers (Chrome, Edge, Brave, Opera). Firefox and Safari do NOT support it. Currently, if a user opens the app in an unsupported browser, the app will crash with a `TypeError: window.showDirectoryPicker is not a function` error with no graceful degradation.

The function `supportsFileSystemAccess()` already exists in `src/lib/filesystem.ts:55-57` but it is **never called** before attempting to use the API. The store's `openFolder` action and `App.tsx` directly call `pickProjectDirectory()` without any prior support check.

**Affected code path:**

* `src/lib/filesystem.ts:59-67` — `pickDirectory()` catches `AbortError` but throws on all other errors including permission-denied and unsupported browser

* `src/lib/store.ts:294-313` — `openFolder()` calls `pickProjectDirectory()` with no prior support check

* `src/App.tsx` — No check on mount before any FSAA usage

**Current behavior:**

* Firefox/Safari user opens kandown.html

* App attempts to call `window.showDirectoryPicker`

* `TypeError` is thrown, app crashes with blank screen

* No error message, no recovery, no indication of why

***

## Why This Needs Fixing

**Severity: Critical**

This is a **crash-level bug** that affects every Firefox and Safari user who tries to open the app. The README explicitly states Firefox and Safari are unsupported, but the app should detect this upfront and show a helpful error message instead of crashing silently.

## How to Fix

### Step 1: Create Error Type Hierarchy (`src/lib/errors.ts`)

Create a new file `src/lib/errors.ts` with a hierarchy of Kandown-specific errors:

```typescript
export class KandownError extends Error {
  constructor(message: string, public readonly code?: string) {
    super(message);
    this.name = 'KandownError';
  }
}

export class BrowserNotSupportedError extends KandownError {
  constructor(browser: string) {
    super(`File System Access API is not supported in ${browser}. Please use Chrome, Edge, Brave, or Opera.`);
    this.name = 'BrowserNotSupportedError';
  }
}

export class PermissionDeniedError extends KandownError {
  constructor(context: string) {
    super(`Permission denied: ${context}`);
    this.name = 'PermissionDeniedError';
  }
}

export class FileNotFoundError extends KandownError {
  constructor(path: string) {
    super(`File not found: ${path}`);
    this.name = 'FileNotFoundError';
  }
}

export class DiskFullError extends KandownError {
  constructor(path: string) {
    super(`Disk full or quota exceeded: cannot write to ${path}`);
    this.name = 'DiskFullError';
  }
}

export class CorruptedDataError extends KandownError {
  constructor(path: string, details?: string) {
    super(`Corrupted data in ${path}${details ? `: ${details}` : ''}`);
    this.name = 'CorruptedDataError';
  }
}

// Result type for operations that can fail without throwing
export type Result<T, E = Error> =
  | { ok: true; value: T }
  | { ok: false; error: E };
```

### Step 4: Ensure `pickDirectory` Uses New Error Types

Update `src/lib/filesystem.ts:pickDirectory` to throw specific errors:

```typescript
export async function pickDirectory(): Promise<FileSystemDirectoryHandle | null> {
  try {
    return await window.showDirectoryPicker({ mode: 'readwrite' });
  } catch (e) {
    const err = e as Error;
    if (err.name === 'AbortError') return null;
    if (err.name === 'NotAllowedError' || err.name === 'SecurityError') {
      throw new PermissionDeniedError('directory access');
    }
    throw err; // Re-throw unknown errors
  }
}
```
