# Context

The agent launch pipeline in the CLI has multiple unguarded operations that can crash the board TUI without useful error messages. The flow is:

1. User selects an agent in the agent picker
2. `board.tsx` calls `launchAgent()`
3. `launcher.ts` reads the task file, writes a context file, moves the task to "In Progress", spawns the agent
4. If any step fails, the TUI shows a generic error or crashes

**Unguarded operations:**

### `launcher.ts:91` — moveTaskToColumn
```typescript
moveTaskToColumn(kandownDir, taskId, 'In Progress'); // No error handling
```
At `board-reader.ts:123-138`, `moveTaskToColumn` uses `writeFileSync` with no try/catch. If the task file is locked, deleted, or the disk is full, this throws uncaught.

### `launcher.ts:95-96` — Context file write
```typescript
const contextFile = join(tmpdir(), `kandown-${taskId}-context.md`);
writeFileSync(contextFile, `${systemPrompt}\n\n---\n\n${taskPrompt}`, 'utf8'); // No try/catch
```
If `/tmp` is full or not writable, this throws uncaught. The agent never launches but the task was already moved to "In Progress".

### `launcher.ts:106-113` — tmux exec
```typescript
execSync(`tmux split-window -h -p 50 ...`, { stdio: 'inherit' }); // Can throw
```
If tmux is not installed or the session name is invalid, this throws.

### `launcher.ts:128-130` — Process exit on agent exit
```typescript
child.on('exit', code => { process.exit(code ?? 0); });
```
If the agent exits with non-zero code (crashed), the TUI exits immediately with no cleanup or error message.

### `board-reader.ts:51-70` — readBoard
```typescript

    const content = readFileSync(filePath, 'utf8'); // Can throw
    // ...

```
If any task file is unreadable, `readBoard` throws and the board TUI shows nothing.

### `board-reader.ts:104-117` — readAgentDoc

## Why This Needs Fixing

**Severity: Medium**

The agent launch flow is a key feature of the TUI. If any step fails, the user gets a blank error or the TUI exits entirely. The task may already be in "In Progress" status even though the agent never started. The fix needs to:
1. Handle each step independently with clear error messages
2. Rollback the task status if the agent fails to start
3. Never crash the TUI — always show a recoverable error

## How to Fix

### Step 1: Guard launcher.ts with Step-by-Step Error Handling

```typescript
export function launchAgent(opts: LaunchAgentOpts): void {
  const { taskId, agentId, kandownDir, onBeforeExec } = opts;
  const agentDef = getAgentById(agentId);
  
  if (!agentDef) {
    throw new Error(`Unknown agent: ${agentId}`);
  }

  // Step 1: Read task file
  let task: ReturnType<typeof readTask>;
  try {
    task = readTask(kandownDir, taskId);
    if (!task) {
      throw new Error(`Task ${taskId} not found`);
    }
  } catch (e) {
    throw new Error(`Failed to read task ${taskId}: ${e.message}`);
  }

  // Step 2: Read agent doc (non-critical — fallback to empty)
  let agentDoc = '';
  try {
    agentDoc = readAgentDoc(kandownDir);
  } catch {
    // Non-critical — agent can still launch without the doc
  }

  // Step 3: Move task to "In Progress" (rollback on failure)
  let taskMoved = false;
  try {
    taskMoved = moveTaskToColumn(kandownDir, taskId, 'In Progress');
    if (!taskMoved) {
      throw new Error('Task file could not be updated');
    }
  } catch (e) {
    throw new Error(`Failed to move task to In Progress: ${e.message}`);
  }

  // Step 4: Write context file (non-critical — agent can still launch)
  try {
    const contextFile = join(tmpdir(), `kandown-${taskId}-context.md`);
    const systemPrompt = agentDoc;
    const taskPrompt = buildPrompt(taskId, task);
    writeFileSync(contextFile, `${systemPrompt}\n\n---\n\n${taskPrompt}`, 'utf8');
  } catch (e) {
    // Context file is optional — warn but don't block agent launch
    console.warn(`Failed to write context file: ${e.message}`);
  }

  // Step 5: Launch agent
  try {
    onBeforeExec?.();
    // ... spawn or tmux
  } catch (e) {
    // Rollback: move task back to original column
    if (taskMoved) {
      try {
        moveTaskToColumn(kandownDir, taskId, task.frontmatter.status || 'Backlog');
      } catch {
        // Rollback failed — at least log it
        console.error(`Failed to rollback task ${taskId} status`);
      }
    }
    throw new Error(`Failed to launch ${agentDef.name}: ${e.message}`);
  }
}
```

### Step 2: Fix moveTaskToColumn Error Handling

In `board-reader.ts`:

```typescript
export function moveTaskToColumn(kandownDir: string, taskId: string, targetColumn: string): boolean {
  const taskPath = join(kandownDir, 'tasks', `${taskId}.md`);
  if (!existsSync(taskPath)) return false;
  
  try {
    const parsed = readTask(kandownDir, taskId);
    if (!parsed) return false;
    
    const updated = {
      ...parsed.frontmatter,
      status: targetColumn,
    };
    writeFileSync(taskPath, serializeTaskFile(updated, parsed.body), 'utf8');
    return true;
  } catch (e) {
    console.error(`Failed to move task ${taskId} to ${targetColumn}:`, e);
    return false;
  }
}
```

### Step 3: Graceful Agent Exit

Instead of `process.exit(code ?? 0)`:

```typescript
child.on('exit', code => {
  if (code !== 0 && code !== null) {
    // Agent exited with error — show message, don't crash TUI
    // In tmux mode, the pane just closes
    // In direct mode, control returns to the user
    if (!isTmux) {
      console.error(`Agent exited with code ${code}`);
      // Don't call process.exit — let the TUI continue
    }
  } else {
    // Agent exited normally
    if (!isTmux) {
      process.exit(0); // Only exit in direct mode on success
    }
  }
});
```
