# Context

Several critical operations use `Promise.all()` to process multiple tasks in parallel. The problem: **`Promise.all` fails fast** — if ANY promise rejects, the entire operation rejects. In a file-based app where each task is an independent file, this means one corrupted/unreadable task can break the entire board load.

**Affected locations:**

### readAllTasks — store.ts:175-190
```typescript
const tasks = await Promise.all(ids.map(id => fsReadTaskFile(tasksDir, id)));
```
If any single task file is unreadable, the ENTIRE board fails to load. All tasks disappear.

### persistColumnOrder — store.ts:192-215
```typescript
await Promise.all(writes);
```
If any task write fails mid-column-change, the column state is inconsistent — some tasks moved, others didn't.

### reloadBoard content loading — store.ts:363-392
```typescript
// Eagerly loads task content for all visible tasks
// If any load fails, the entire reload fails
```

### Watcher tick — watcher.ts:84-111
```typescript
// Checks each task sequentially in a for loop
// If one throws, remaining tasks are skipped
```

**The fix:** Replace `Promise.all` with `Promise.allSettled` in all locations where individual failures should be tolerated. Aggregate results, report failures, but don't let one bad file take down the whole operation.

## How to Fix

### Step 1: Fix readAllTasks

```typescript
export async function readAllTasks(
  tasksDir: FileSystemDirectoryHandle,
  ids: string[]
): Promise<{
  tasks: ParsedTask[];
  failedIds: Array<{ id: string; error: Error }>;
}> {
  const settled = await Promise.allSettled(
    ids.map(async id => {
      const result = await fsReadTaskFile(tasksDir, id);
      return { id, result };
    })
  );

  const tasks: ParsedTask[] = [];
  const failedIds: Array<{ id: string; error: Error }> = [];

  settled.forEach((result, index) => {
    const id = ids[index];
    if (result.status === 'fulfilled') {
      if (result.value.result) {
        tasks.push(result.value.result);
      }
      // null result means file not found — skip silently
    } else {
      failedIds.push({ id, error: result.reason });
    }
  });

  return { tasks, failedIds };
}
```

### Step 2: Fix persistColumnOrder

```typescript
async function persistColumnOrder(
  tasksDir: FileSystemDirectoryHandle,
  affected: Array<{ id: string; status: string; order: number }>,
  columns: BoardColumn[]
): Promise<{ succeeded: string[]; failed: Array<{ id: string; error: Error }> }> {
  const settled = await Promise.allSettled(
    affected.map(async task => {
      const content = await fsReadTaskFile(tasksDir, task.id);
      if (!content) throw new Error(`Task ${task.id} not found`);
      const updated = { ...content.frontmatter, status: task.status, order: task.order };
      await fsWriteTaskFile(tasksDir, task.id, updated, content.body);
      return task.id;
    })
  );

  const succeeded: string[] = [];
  const failed: Array<{ id: string; error: Error }> = [];

  settled.forEach((result, index) => {
    const task = affected[index];
    if (result.status === 'fulfilled') {
      succeeded.push(task.id);
    } else {
      failed.push({ id: task.id, error: result.reason });
    }
  });

  return { succeeded, failed };
}
```

```typescript
const { succeeded, failed } = await persistColumnOrder(tasksDirHandle, affected, config.board.columns);

  // Optionally: reload board to sync with actual file state

}
```
