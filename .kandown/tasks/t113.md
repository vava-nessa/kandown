# Context

The CLI (`bin/kandown.js`) has two areas with insufficient error handling:

### 1. copyRecursive — Init Command Crash

`bin/kandown.js:95-106` — The `copyRecursive` function used during `kandown init` has no error handling:

```javascript
function copyRecursive(src, dest) {
  if (!existsSync(src)) return;
  mkdirSync(dest, { recursive: true });
  for (const entry of readdirSync(src)) {
    const srcPath = join(src, entry);
    const destPath = join(dest, entry);
    if (statSync(srcPath).isDirectory()) {  // Can throw on permission error
      copyRecursive(srcPath, destPath);
    } else {
      copyFileSync(srcPath, destPath);  // Can throw on disk full, permission denied
    }
  }
}
```

If any file copy fails mid-init:
- `.kandown/` is partially created
- Some templates are copied, others are not
- The error propagates and `init` crashes
- No cleanup of partial installation
- Running `init` again may fail or produce inconsistent state

### 2. HTTP Server — Port Error Handling

`bin/kandown.js:386-405` — `listenOnAvailablePort` only handles `EADDRINUSE`:

```javascript
for (let port = startPort; port <= endPort; port++) {
  const server = createServeServer(kandownDir);
  try {
    await listen(server, port);
    return { server, port };
  } catch (e) {
    if (e.code !== 'EADDRINUSE') throw e;  // EACCES, EPERM → crash
  }
}
```

`EACCES` (permission denied, usually ports below 1024) and other errors crash instead of trying the next port.

## How to Fix

### Step 1: Resilient copyRecursive

```javascript
function copyRecursive(src, dest) {
  if (!existsSync(src)) return { success: true, errors: [] };
  
  const errors = [];
  mkdirSync(dest, { recursive: true });
  
  let entries;
  try {
    entries = readdirSync(src);
  } catch (e) {
    return { success: false, errors: [`Failed to read ${src}: ${e.message}`] };
  }

  for (const entry of entries) {
    const srcPath = join(src, entry);
    const destPath = join(dest, entry);
    
    try {
      if (statSync(srcPath).isDirectory()) {
        const result = copyRecursive(srcPath, destPath);
        errors.push(...result.errors);
      } else {
        copyFileSync(srcPath, destPath);
      }
    } catch (e) {
      errors.push(`Failed to copy ${srcPath}: ${e.message}`);
    }
  }

  return { success: errors.length === 0, errors };
}
```

In `cmdInit`, check the result:

```javascript
const result = copyRecursive(templatesDir, kandownDir);
if (result.errors.length > 0) {
  warn(`Some files could not be copied:`);
  result.errors.forEach(e => warn(`  - ${e}`));
  if (!result.success) {
    err('Init failed — too many files could not be copied');
    process.exit(1);
  }
}
```

### Step 2: Handle More Error Codes in Port Selection

```javascript
for (let port = startPort; port <= endPort; port++) {
  const server = createServeServer(kandownDir);
  try {
    await listen(server, port);
    return { server, port };
  } catch (e) {
    if (e.code === 'EADDRINUSE' || e.code === 'EACCES') {
      continue; // Try next port
    }
    throw e; // Unexpected error
  }
}
```
