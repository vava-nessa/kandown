# Context

The FileWatcher (`src/lib/watcher.ts`) polls for file changes and fires callbacks in the store (`src/lib/store.ts:876-925`). Multiple watcher event handlers have **no try/catch**, meaning any error in these callbacks silently kills the watcher's event processing. The user sees no board updates until they manually reload.

**Affected code:**

### Watcher configChanged handler (`store.ts:876-879`)
```typescript
configChanged: () => {
  get().loadConfig();  // No try/catch — if loadConfig throws, callback dies
  get().toast('Config updated from disk', 'info');
}
```


## Why This Needs Fixing

**Severity: High**

When a watcher callback throws (due to permission loss, disk error, or corrupted file), the entire polling mechanism silently stops. The user's board becomes stale — changes made externally (by AI agents, git operations, other tools) are never reflected. The user has no indication that the watcher is broken until they manually reload.

This is particularly problematic in the AI-agent workflow where agents modify task files and the user expects to see changes in real-time.

## How to Fix

### Step 1: Wrap Store Watcher Callbacks

In `src/lib/store.ts`, wrap all watcher callbacks:

```typescript
const watcher = createFileWatcher({
  configChanged: () => {
    try {
      get().loadConfig();
      get().toast('Config updated from disk', 'info');
    } catch (e) {
      console.error('[Watcher] configChanged error:', e);
      get().toast('Failed to reload config from disk', 'warning');
    }
  },

  taskChanged: async (taskId: string) => {
    try {
      const { tasksDirHandle, config, taskContents } = get();
      if (!tasksDirHandle) return;
      const content = await fsReadTaskFile(tasksDirHandle, taskId);
      if (!content) return;
      // ... existing update logic
    } catch (e) {
      console.error(`[Watcher] taskChanged error for ${taskId}:`, e);
      // Don't toast for every task change failure — too noisy
      // Just skip this task and continue
    }
  },

  newTaskDetected: async (taskId: string) => {
    try {
      const { tasksDirHandle } = get();
      if (!tasksDirHandle) return;
      const content = await fsReadTaskFile(tasksDirHandle, taskId);
      if (!content) return;
      // ... existing add logic
    } catch (e) {
      console.error(`[Watcher] newTaskDetected error for ${taskId}:`, e);
      get().toast(`New task ${taskId} detected but could not be loaded`, 'warning');
    }
  },
});
```

### Step 2: Protect FileWatcher tick()

In `src/lib/watcher.ts`:

```typescript
private async tick(): Promise<void> {
  // Protect entire tick from aborting
  try {
    // Check config hash
    const configText = await readConfigFileText(this.kandownHandle);
    // ... existing config check logic

    // Check each task — protect individually
    for (const taskId of [...this.knownTaskIds]) {
      try {
        const taskText = await readTaskFileText(this.tasksDirHandle, taskId);
        if (taskText !== null) {
          const hash = await this.hash(taskText);
          if (this.taskHashes.get(taskId) !== hash) {
            this.taskHashes.set(taskId, hash);
            this.emitter.emit('taskChanged', taskId);
          }
        } else {
          // File was deleted or became unreadable
          // Remove from known IDs so we don't keep polling a dead file
          this.knownTaskIds.delete(taskId);
          this.taskHashes.delete(taskId);
        }
      } catch (e) {
        console.error(`[Watcher] Error checking task ${taskId}:`, e);
        // Continue to next task — don't abort tick
      }
    }

    // Sync task dir
    await this.syncTaskDir(true);
  } catch (e) {
    console.error('[Watcher] Tick error:', e);
    this.consecutiveErrors++;
    
    if (this.consecutiveErrors >= 5) {
      this.stop();
      this.emitter.emit('watcherError', 'File watcher stopped due to repeated errors');
    }
  }
}
```

### Step 3: Add Error Counter and Auto-Disable

```typescript
class FileWatcher {
  private consecutiveErrors = 0;
  private maxConsecutiveErrors = 5;

  // In tick success:
  this.consecutiveErrors = 0; // Reset on successful tick

  // In tick failure:
  this.consecutiveErrors++;
  if (this.consecutiveErrors >= this.maxConsecutiveErrors) {
    this.stop();
    this.emitter.emit('watcherError', `Watcher disabled after ${this.maxConsecutiveErrors} errors`);
  }
}
```

### Step 4: Add Watcher Status to Store

```typescript
interface StoreState {
  watcherActive: boolean;
  watcherError: string | null;
}

// Listen for watcher errors:
watcher.on('watcherError', (msg) => {
  set({ watcherActive: false, watcherError: msg });
  get().toast(msg, 'error', 10000);
});
```
