# Context

The web app has **no global error handlers**. There are no `window.onerror` or `window.addEventListener('unhandledrejection')` handlers anywhere in the codebase. Uncaught errors and unhandled promise rejections go directly to the browser console, invisible to the user.

**Current state:**
- No `window.onerror` handler
- No `window.addEventListener('unhandledrejection')` handler
- Unhandled rejections silently accumulate in the console
- No centralized error logging
- No way to track how often errors occur in production

**Known sources of unhandled rejections:**

1. `store.ts:930-932` — `listRecentProjects().then(...)` with no `.catch()`
2. `App.tsx:50-56` — `changeLanguage` called with `void`, rejection unhandled
3. `App.tsx:59-68` — `openRecentProject(match)` called without `.catch()`
4. `App.tsx:69-117` — store actions called from keyboard handler, async rejections unhandled
5. `Drawer.tsx:58-60` — `saveDrawerMetadata` rejection unhandled
6. `Drawer.tsx:79-82` — `flushAutoSave`/`saveDrawer` rejections unhandled
7. `SettingsPage.tsx:533` — `loadConfig()` rejection in file watcher callback
8. `CommandPalette.tsx` — `loadTaskContents`, `openDrawer`, `createTask` all fire-and-forget
9. `MarkdownEditor.tsx:76` — `marked.parse(value)` returns Promise but not awaited

These are scattered throughout the codebase. While individual fixes are covered in other tasks (t100-t114), this task adds the **safety net** that catches anything missed.

## Why This Needs Fixing

**Severity: High (Safety Net)**

Even after fixing all known error sites, new unhandled rejections can appear from:
- Browser API changes
- Edge cases in third-party libraries (marked, etc.)
- Future code changes that introduce new async operations
- Race conditions in complex state interactions

A global handler is the last line of defense. It should:
1. Never crash the app
2. Provide useful debugging info
3. Not spam the user with error toasts

## How to Fix

### Step 1: Create Global Error Handler Module

Create `src/lib/globalErrors.ts`:

```typescript
const ERROR_THROTTLE_MS = 5000;
const MAX_TOAST_ERRORS = 3;

let lastErrorTime = 0;
let errorCount = 0;
let toastCount = 0;

function shouldShowToast(): boolean {
  const now = Date.now();
  if (now - lastErrorTime < ERROR_THROTTLE_MS) return false;
  if (toastCount >= MAX_TOAST_ERRORS) return false;
  lastErrorTime = now;
  toastCount++;
  return true;
}

function formatError(source: string, error: unknown): string {
  if (error instanceof Error) {
    return `[${source}] ${error.name}: ${error.message}`;
  }
  return `[${source}] ${String(error)}`;
}

export function setupGlobalErrorHandlers(): void {
  // Catch synchronous errors
  window.addEventListener('error', (event: ErrorEvent) => {
    console.error('[Global] Uncaught error:', event.error);
    errorCount++;

    if (shouldShowToast()) {
      // Import store lazily to avoid circular dependency
      import('./store').then(({ useStore }) => {
        useStore.getState().toast('An unexpected error occurred', 'warning', 5000);
      });
    }

    // Don't prevent default — let browser log to console too
  });

  // Catch unhandled promise rejections
  window.addEventListener('unhandledrejection', (event: PromiseRejectionEvent) => {
    console.error('[Global] Unhandled rejection:', event.reason);
    errorCount++;

    if (shouldShowToast()) {
      import('./store').then(({ useStore }) => {
        useStore.getState().toast('An async operation failed unexpectedly', 'warning', 5000);
      });
    }

    // Prevent default browser console warning
    event.preventDefault();
  });

  // Reset toast counter every 30 seconds
  setInterval(() => {
    toastCount = 0;
  }, 30000);
}

export function getErrorStats(): { errorCount: number } {
  return { errorCount };
}
```

### Step 2: Call Setup Before React Mounts

In `src/main.tsx`:

```typescript
import { setupGlobalErrorHandlers } from './lib/globalErrors';

// Set up global error handlers BEFORE React mounts
setupGlobalErrorHandlers();

ReactDOM.createRoot(document.getElementById('root')!).render(
  <App />
);
```

### Step 3: Add Development Mode Error Details

In development, show more detailed error information:

```typescript
if (import.meta.env.DEV) {
  window.addEventListener('unhandledrejection', (event) => {
    console.group('[DEV] Unhandled Promise Rejection');
    console.error('Reason:', event.reason);
    console.trace();
    console.groupEnd();
  });
}
```
