  ## Root cause
  Two systemic issues plus an orphan CSS layer:
  1. `Card.tsx` and `CardStack.tsx` hardcoded `bg-white dark:bg-white/90` —
     white in BOTH modes. Combined with `text-fg` (white in dark) → white-on-white.
  2. `Header.tsx` hardcoded `dark:bg-[#1a1a1a]/80` and `bg-white dark:bg-white/20`
     for the view toggle, ignoring the active skin.
  3. `globals.css :root` defined `--muted: 0 0% 43%` (a foreground tone) AND a
     stray legacy `.light-skin` block that double-shadowed the active tokens. It
     redefined `--border` etc. as rgba and never matched what `applyProjectTheme`
     applies. `--muted-foreground` was missing from the bootstrap defaults.
  4. BlockNote's data-color-scheme attribute was only set on the floating portal,
     not on the editor wrapper itself, so Mantine code-block CSS (hardcoded
     `rgb(22 22 22)` bg + white text) never picked up dark mode.

  ## Changes
  - src/components/Card.tsx
    → main card surface: `bg-card border-border` (semantic, theme-aware)
    → delete button: `bg-card/80 border-border`
  - src/components/CardStack.tsx
    → main + 2 shadow layers use `bg-card` / `bg-card/40` / `bg-card/60`
  - src/components/Header.tsx
    → header bar: `bg-card/80 border-border` (drops `bg-[#1a1a1a]/80`)
    → search input: `bg-secondary/60` + `focus-within:bg-secondary`
    → view toggle active state: `bg-card text-fg shadow-sm`
  - src/styles/globals.css
    → removed legacy `--app-bg`, `--surface`, `--text`, `--col-*`, `--purple/blue/amber/green`
    → fixed `--muted` (was 43%, now 11%) and added `--muted-foreground: 0 0% 43%`
    → bootstrapped `--border / --border-strong / --border-focus` as HSL triples
    → deleted the dead `.light-skin` block (theming runs through `applyProjectTheme`)
    → strengthened `.bn-block-content[data-content-type="codeBlock"]` rules:
      forces `background: hsl(var(--muted))`, `color: hsl(var(--foreground))`,
      transparent inner `<pre>`, and themed `<select>` to defeat BlockNote's
      hardcoded `rgb(22 22 22)` + white text.
  - src/components/ui/BlockNoteMarkdownEditor.tsx
    → tracks `colorScheme` via MutationObserver on `html.dark`

## Problème

En mode dark, deux problèmes visuels critiques :

1. **Editor Markdown** : les code blocks rendent mal (fond blanc ou illisible)
2. **Cards** : les cartes passent en blanc en dark mode → texte blanc sur fond blanc = illisible

## Objectif

Audit complet du mode dark :

- Vérifier TOUS les composants en mode dark (Board, Column, Card, CardStack, Drawer, Header, CommandPalette, ListView, SettingsPage, EmptyState)
- Repérer les cas blanc-sur-blanc et noir-sur-noir
- Corriger les fonds, textes, bordures, badges, icônes
- Tester les 5 skins en mode dark
- Tester les backgrounds (solid, static-gradient)
- Vérifier l'editor BlockNote en dark mode (fond code blocks, syntax highlighting)
