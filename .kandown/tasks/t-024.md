## Context

Actuellement `kandown` (sans args) ouvre `kandown.html` via `file://` avec `open`/`xdg-open`. Ça force l'utilisateur à sélectionner manuellement son dossier de projet à chaque fois via la File System Access API du navigateur.

Le but est de remplacer ce comportement par un mini-serveur HTTP Node.js natif (zéro dépendance npm) qui sert `kandown.html` sur `http://localhost:PORT` et injecte le chemin du projet dans la page. Le web app détectera le mode serveur et s'initialisera automatiquement — sans folder picker.

**Contrainte forte :** Zéro nouvelle dépendance npm. Uniquement `http`, `fs`, `path`, `url` natifs.

**Cette tâche couvre uniquement le serveur dans `bin/kandown.js`.** Les modifications du web app sont dans t-026 et t-027.

## Subtasks

- [ ] Créer `cmdServe(rawArgs)` dans `bin/kandown.js` — remplace l'actuel `case undefined:` (openInBrowser + cmdTui)
- [ ] Implémenter le serveur avec `http.createServer()` — pas de framework
- [ ] `GET /` → lire `kandown.html`, injecter `<script>window.__KANDOWN_ROOT__ = "...";</script>` avant `</head>`, servir
- [ ] Trouver un port libre : tester 2048, si occupé essayer 2049…2060, abandonner avec message si tout pris
- [ ] Support `--port N` dans `parseArgs()` pour custom port
- [ ] Après `server.listen()` : appeler `openInBrowser('http://localhost:PORT')` puis `cmdTui('board', rest)`
- [ ] Logger : `✓ Web UI: http://localhost:PORT` et `→ Project: /chemin/.kandown`
- [ ] Graceful shutdown : `SIGINT`/`SIGTERM` → `server.close()` avant `process.exit(0)`
- [ ] `GET|PUT|DELETE /api/*` → stub `handleApi()` qui retourne `501 Not Implemented` (implémenté dans t-025)
- [ ] Tout le reste → 404

## Spec : Injection KANDOWN_ROOT

```js
const html = readFileSync(htmlPath, 'utf8');
const injected = html.replace(
  '</head>',
  `<script>window.__KANDOWN_ROOT__ = ${JSON.stringify(kandownDir)};</script>\n</head>`
);
```

## Spec : Routage

```js
const server = http.createServer((req, res) => {
  const url = new URL(req.url, 'http://localhost');
  if (req.method === 'OPTIONS') return handleCors(res);
  if (url.pathname === '/') return serveApp(res, kandownDir);
  if (url.pathname.startsWith('/api/')) return handleApi(req, res, url, kandownDir);
  res.writeHead(404); res.end('Not found');
});
```

## Spec : Headers CORS (toutes les réponses /api/*)

```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, PUT, DELETE, OPTIONS
Access-Control-Allow-Headers: Content-Type
```
