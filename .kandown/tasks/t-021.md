Créer l'écran principal du TUI kandown : un kanban board navigable au clavier avec visualisation des colonnes, sélection de tasks, vue détail, et lancement d'agents AI.

## Context

Le TUI existant (Ink 7) a un écran settings (`src/cli/screens/settings.tsx`). On ajoute un écran `board` qui affiche les 5 colonnes du kanban avec les tasks, permet la navigation clavier, l'ouverture de détails, et le lancement d'agents via un picker overlay.

C'est le fichier le plus gros du projet (~300-400 lignes).

## Subtasks

- [ ] Créer `src/cli/screens/board.tsx` — layout colonnes adaptatif (terminal width), curseur colonne/row, affichage tasks tronquées
- [ ] Implémenter la navigation clavier (h/l ou ←/→ colonnes, j/k ou ↑/↓ tasks, Enter détail, q quit)
- [ ] Implémenter la vue détail task (affiche contenu complet du fichier task, subtasks, context)
- [ ] Créer `src/cli/screens/agent-picker.tsx` — modal overlay, liste agents installés, sélection + lancement
- [ ] Wiring dans `src/cli/app.tsx` — ajouter `case 'board'` au router

## Notes

- Layout : 5 colonnes en `<Box flexDirection="row">`, largeur calculée depuis `process.stdout.columns`
- Sur terminal étroit (<100 cols) : tronquer les titres agressivement, ou masquer colonnes vides
- Le pattern de navigation est le même que settings.tsx : `useInput` de Ink + state avec useState
- L'agent picker ne montre que les agents détectés comme installés (via agents.ts)
- Le board se recharge depuis le disque avec `r` (utile si un agent modifie board.md pendant qu'on regarde)
