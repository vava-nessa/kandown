# Context

When a task file is corrupted, unreadable (permission revoked), or otherwise fails to read, `readTaskFile()` in `src/lib/filesystem.ts:147-166` catches ALL errors and returns a ghost task — an empty task with just the ID and default values. This silently hides data corruption and makes it impossible for the user to know their task file is broken.

**Current behavior:**
```typescript
export async function readTaskFile(tasksDir: FileSystemDirectoryHandle, id: string) {
  try {
    const h = await tasksDir.getFileHandle(`${id}.md`);
    const file = await h.getFile();
    const text = await file.text();
    return parseTaskFile(text);
  } catch {
    // ALL errors (file not found, permission denied, disk error, corrupted content)
    // → return identical ghost task with no indication of what went wrong
    return {
      frontmatter: { id, title: '', status: 'Backlog', order: 0 } as TaskFrontmatter,
      body: `# ${id}\n\n## Context\n\n\n## Subtasks\n\n`,
    };
  }
}
```

**Problems:**
1. `file not found` — this is OK to return ghost (task was deleted externally)
2. `permission denied / revoked` — user needs to know, task is accessible but not readable
3. `disk read error` — transient, may succeed on retry
4. `corrupted content / parse error` — task exists but is unreadable, data may be recoverable
5. `file deleted mid-read` — race condition, may need retry

Current code treats all 5 cases identically: returns a ghost task and shows nothing to the user.

## Subtasks

- [ ] Create distinct error types for file read failures
- [ ] Distinguish "file not found" from "file unreadable" in readTaskFile
- [ ] Return `Result<T, Error>` instead of ghost tasks for unreadable files
- [ ] Update `readAllTasks` in store.ts to handle partial failures gracefully
- [ ] Show toast/warning when a task file is corrupted or unreadable
- [ ] Add "corrupted tasks" list to board state for user visibility
- [ ] Add retry option for transient file read errors
- [ ] Update `reloadBoard` to preserve previous board state on failure

## Why This Needs Fixing

**Severity: Critical**

A corrupted task file is silently replaced with an empty ghost task. The user loses their actual task content (context, subtasks, notes, body) without any warning. The ghost task has the same ID, so the user may not even realize their data is gone until they reload and see empty content.

This can happen when:
- Browser loses permission to read the file (user revokes in OS)
- Disk error during read (transient)
- File is corrupted by a partial write
- File is deleted by another process during iteration

## How to Fix

### Step 1: Return Result Types from read operations

Instead of returning ghost tasks, return explicit success/failure:

```typescript
// src/lib/filesystem.ts

export type FileReadResult<T> =
  | { ok: true; value: T }
  | { ok: false; error: FileReadError; isFileNotFound: boolean };

export class FileReadError extends Error {
  constructor(
    message: string,
    public readonly code: 'NOT_FOUND' | 'PERMISSION_DENIED' | 'DISK_ERROR' | 'PARSE_ERROR' | 'UNKNOWN'
  ) {
    super(message);
    this.name = 'FileReadError';
  }
}

export async function readTaskFile(
  tasksDir: FileSystemDirectoryHandle,
  id: string
): Promise<FileReadResult<ParsedTask>> {
  try {
    const h = await tasksDir.getFileHandle(`${id}.md`);
    const file = await h.getFile();
    const text = await file.text();
    
    // Check for empty file (might be ghost from previous error)
    if (!text || text.trim() === '') {
      return { ok: false, error: new FileReadError(`Task file ${id}.md is empty`, 'PARSE_ERROR'), isFileNotFound: false };
    }
    
    return { ok: true, value: parseTaskFile(text) };
  } catch (e) {
    if (e instanceof DOMException) {
      if (e.name === 'NotFoundError') {
        return { ok: false, error: new FileReadError(`Task ${id}.md not found`, 'NOT_FOUND'), isFileNotFound: true };
      }
      if (e.name === 'NotAllowedError' || e.name === 'SecurityError') {
        return { ok: false, error: new FileReadError(`Permission denied for ${id}.md`, 'PERMISSION_DENIED'), isFileNotFound: false };
      }
    }
    return { ok: false, error: new FileReadError(`Failed to read ${id}.md: ${e}`, 'UNKNOWN'), isFileNotFound: false };
  }
}
```

### Step 2: Update readAllTasks to aggregate results

In `src/lib/store.ts` or `src/lib/filesystem.ts`:

```typescript
export async function readAllTasks(
  tasksDir: FileSystemDirectoryHandle,
  ids: string[]
): Promise<{ tasks: ParsedTask[]; failedIds: Array<{ id: string; error: FileReadError }> }> {
  const results = await Promise.all(
    ids.map(async id => {
      const result = await readTaskFile(tasksDir, id);
      return { id, result };
    })
  );

  const tasks: ParsedTask[] = [];
  const failedIds: Array<{ id: string; error: FileReadError }> = [];

  for (const { id, result } of results) {
    if (result.ok) {
      tasks.push(result.value);
    } else if (result.isFileNotFound) {
      // File was deleted — skip silently (task was removed externally)
      continue;
    } else {
      // File exists but couldn't be read — record as failed
      failedIds.push({ id, error: result.error });
    }
  }

  return { tasks, failedIds };
}
```

### Step 3: Update reloadBoard to preserve state and show warnings

```typescript
reloadBoard: async () => {
  const { tasksDirHandle, config, columns, taskContents } = get();
  if (!tasksDirHandle) return;

  // Capture previous state for rollback on complete failure
  const previousColumns = columns;
  const previousTaskContents = taskContents;

  try {
    const taskIds = await listTaskIds(tasksDirHandle);
    const { tasks, failedIds } = await readAllTasks(tasksDirHandle, taskIds);

    // Show warning for failed tasks
    if (failedIds.length > 0) {
      const msg = failedIds.length === 1 
        ? `Task ${failedIds[0].id} could not be loaded (${failedIds[0].error.code})`
        : `${failedIds.length} tasks could not be loaded`;
      get().toast(msg, 'warning', 8000);
      
      // Store failed task IDs for user visibility
      set({ failedTaskIds: failedIds.map(f => f.id) });
    }

    // Build board from readable tasks only
    const parsedTasks = tasks.map(task => taskToBoardTask(task));
    const newColumns = buildColumnsFromTasks(parsedTasks, config.board.columns);
    
    set({ boardTitle: 'Project Kanban', columns: newColumns, failedTaskIds: [] });
    
    // Load content for readable tasks
    const nextContents = new Map<string, TaskContent>();
    for (const t of parsedTasks) {
      nextContents.set(t.id, { frontmatter: t.frontmatter, body: tasks.find(tk => tk.id === t.id)?.body || '' });
    }
    set({ taskContents: nextContents });

  } catch (e) {
    // On unexpected error (all tasks failed to read), preserve previous state
    get().toast(`Failed to reload board: ${e.message}`, 'error');
    // DO NOT clear columns — keep showing previous board
    // This prevents the "empty board on error" problem
  }
}
```
