# Context

When saving a task in the drawer fails (disk full, permission denied, write error), the drawer stays open with the user's edits intact. However, if the user then **closes the drawer** (clicks X or presses Escape), their unsaved edits are **permanently lost** with no way to recover.

**Current behavior** (`src/lib/store.ts:643-667`):
```typescript
saveDrawer: async () => {
  // ...
  try {
    await fsWriteTaskFile(tasksDirHandle, drawerTaskId, fm, fullBody);
    get().toast('Saved');
    set({ drawerTaskId: null, drawerData: null }); // Close drawer on success
  } catch (e) {
    get().toast('Failed to save: ' + (e as Error).message, 'error');
    // Drawer NOT closed — stays open with edits
    // BUT: if user closes manually, edits are GONE
  }
}
```

**The problem chain:**
1. User edits a task in the drawer
2. Save fails (disk full)
3. Toast says "Failed to save"
4. User closes drawer (X button, Escape, click outside)
5. `handleClose` in Drawer.tsx:79-82 calls `flushAutoSave` then `saveDrawer` 
6. If `flushAutoSave`/`saveDrawer` fails again, drawer closes anyway
7. `drawerData` is set to null
8. User's edits are permanently lost — not in the file, not in the store

## Why This Needs Fixing

**Severity: Medium**

Losing user edits is always a bad experience. In a task management tool where notes and context are valuable, losing a detailed task description due to a transient write failure is particularly painful. The fix needs to:
1. Prevent accidental data loss
2. Give the user clear options (retry, discard, or save elsewhere)
3. Offer recovery even if the drawer was force-closed

## How to Fix

### Step 1: Add Unsaved Changes Tracking

```typescript
interface StoreState {
  // ...
  hasUnsavedDrawerEdits: boolean;
  lastSaveError: string | null;
  drawerRecoveryData: Map<string, DrawerData>; // Recovery buffer per task ID
}
```

### Step 2: Prevent Close with Unsaved Edits

In `Drawer.tsx`, intercept close when there are unsaved edits:

```tsx
const handleClose = async () => {
  const { hasUnsavedDrawerEdits, lastSaveError } = useStore.getState();
  
  if (hasUnsavedDrawerEdits || lastSaveError) {
    // Show confirmation dialog
    const shouldDiscard = await showDiscardConfirmation();
    if (!shouldDiscard) return; // Don't close
    
    // Store in recovery buffer before closing
    const { drawerTaskId, drawerData } = useStore.getState();
    if (drawerTaskId && drawerData) {
      useStore.setState(state => {
        const recovery = new Map(state.drawerRecoveryData);
        recovery.set(drawerTaskId, drawerData);
        return { drawerRecoveryData: recovery };
      });
    }
  }
  
  // Safe to close
  useStore.setState({ 
    drawerTaskId: null, 
    drawerData: null, 
    hasUnsavedDrawerEdits: false,
    lastSaveError: null,
  });
};
```

### Step 3: Add Retry Save Button

In `Drawer.tsx` footer, show a "Retry Save" button when `lastSaveError` is set:

```tsx
{lastSaveError && (
  <div className="flex items-center gap-2 px-4 py-2 bg-destructive/10 border-t border-destructive/30">
    <span className="text-destructive text-sm flex-1">{lastSaveError}</span>
    <button onClick={handleSave} className="px-3 py-1 bg-primary text-primary-foreground rounded text-sm">
      Retry Save
    </button>
  </div>
)}
```

### Step 4: Recovery on Re-open

When opening a drawer for a task that has recovery data:

```typescript
openDrawer: async (taskId: string) => {
  // ... load task from file
  
  // Check for recovery data
  const recovery = get().drawerRecoveryData.get(taskId);
  if (recovery) {
    // Offer to restore unsaved edits
    // Could show a comparison or just a "Restore unsaved edits?" prompt
    const shouldRestore = true; // For now, auto-restore
    if (shouldRestore) {
      set({ drawerData: recovery });
      // Clear recovery data
      const newRecovery = new Map(get().drawerRecoveryData);
      newRecovery.delete(taskId);
      set({ drawerRecoveryData: newRecovery, hasUnsavedDrawerEdits: true });
    }
  }
}
```
