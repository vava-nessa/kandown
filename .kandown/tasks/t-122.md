# Refactor UI components to Shadcn syntax (Buttons, Inputs, Modals)

## Context

The user requested to migrate internal custom UI components towards Shadcn quality. First asked for Buttons, then Inputs and Modals.

## Subtasks

- [x] Migrate Buttons to Shadcn quality (`Button` and `globals.css`)
- [x] Migrate Inputs and Textarea to Shadcn quality (`Input`, `Textarea`, `globals.css`)
- [x] Update `ConflictModal.tsx` custom framed buttons

## What was done

- **Dependencies**: Installed `@radix-ui/react-slot`, `clsx`, `tailwind-merge`, `class-variance-authority`.
- **Utils**: Configured `src/lib/utils.ts` and `tailwind.config.js` (`input`/`ring` colors).
- **Buttons**: Created `src/components/ui/button.tsx`, refactored `KbdButton.tsx`, and globally updated `.btn-base` variants in `globals.css` so standard app buttons perfectly replicate Shadcn styling effortlessly.
- **Inputs & Modals**: Created `input.tsx` and `textarea.tsx`. Updated `.field-input` in `globals.css` to replicate Shadcn inputs. This applies Shadcn visual quality to textareas and inputs throughout the app (`Drawer`, `FilterBar`, `Settings`). Replaced buttons in `ConflictModal.tsx` to `<Button>`.
- Build tested successfully after fixing missing tailwind palette mappings.
