# Context

The web app has **no React Error Boundary** anywhere. If any component throws an error during render (e.g., a malformed task causes a crash in `Board`, `Column`, or `Drawer`), React's default error handling unmounts the entire component tree and replaces it with a blank screen or the browser's error overlay. The user must refresh the page manually.

React error boundaries catch errors in child component trees and display a fallback UI instead of crashing the entire app. They also allow logging of errors for debugging.

**Current behavior:**
- Open a project with a task that has malformed data
- That task's component throws during render
- React unmounts the entire App component
- User sees blank white page (or browser's "error occurred" overlay in some browsers)
- User must manually refresh to recover

**What should happen:**
- React catches the error at the boundary
- App shows "Something went wrong" message with a retry button
- App remains functional for other operations
- Error is logged for debugging

## Why This Needs Fixing

**Severity: Critical**

Without an error boundary, any unexpected error in rendering crashes the entire app with no recovery. This is especially important in a file-based app where task data comes from user files and could be malformed (e.g., JSON parse issues, unexpected data shapes, circular references in state).

The goal is: **the app should never crash hard — it should always degrade gracefully and offer a recovery path**.

## How to Fix

### Step 1: Create ErrorBoundary Component

Create `src/components/ErrorBoundary.tsx`:

```tsx
import { Component, ReactNode, ErrorInfo } from 'react';

interface Props {
  children: ReactNode;
  fallback?: ReactNode;
  onError?: (error: Error, errorInfo: ErrorInfo) => void;
}

interface State {
  hasError: boolean;
  error: Error | null;
  errorInfo: ErrorInfo | null;
}

export class ErrorBoundary extends Component<Props, State> {
  constructor(props: Props) {
    super(props);
    this.state = { hasError: false, error: null, errorInfo: null };
  }

  static getDerivedStateFromError(error: Error): Partial<State> {
    return { hasError: true, error };
  }

  componentDidCatch(error: Error, errorInfo: ErrorInfo) {
    this.setState({ errorInfo });
    this.props.onError?.(error, errorInfo);
    console.error('[ErrorBoundary] Caught error:', error, errorInfo);
  }

  handleRetry = () => {
    this.setState({ hasError: false, error: null, errorInfo: null });
  };

  handleReportCopy = () => {
    const { error, errorInfo } = this.state;
    const report = `
Kandown Error Report
====================
Time: ${new Date().toISOString()}
Error: ${error?.toString() || 'Unknown'}
Stack: ${error?.stack || 'No stack trace'}
Component Stack: ${errorInfo?.componentStack || 'No component stack'}
`.trim();
    navigator.clipboard.writeText(report);
    // Could add toast here: "Error report copied to clipboard"
  };

  render() {
    if (this.state.hasError) {
      if (this.props.fallback) {
        return this.props.fallback;
      }
      return (
        <div className="flex flex-col items-center justify-center min-h-screen bg-background text-foreground p-8">
          <div className="max-w-lg w-full bg-destructive/10 border border-destructive/30 rounded-lg p-6">
            <h2 className="text-xl font-semibold text-destructive mb-2">Something went wrong</h2>
            <p className="text-muted-foreground mb-4">
              An unexpected error occurred. The app is still running but this view crashed.
            </p>
            {this.state.error && (
              <details className="mb-4 p-3 bg-muted rounded text-sm font-mono overflow-auto">
                <summary className="cursor-pointer font-semibold mb-1">Error details</summary>
                <pre className="text-xs whitespace-pre-wrap break-all">{this.state.error.toString()}</pre>
              </details>
            )}
            <div className="flex gap-3">
              <button
                onClick={this.handleRetry}
                className="px-4 py-2 bg-primary text-primary-foreground rounded hover:opacity-90"
              >
                Retry
              </button>
              <button
                onClick={this.handleReportCopy}
                className="px-4 py-2 bg-secondary text-secondary-foreground rounded hover:opacity-90"
              >
                Copy Error Report
              </button>
            </div>
          </div>
        </div>
      );
    }

    return this.props.children;
  }
}
```

### Step 2: Wrap App in ErrorBoundary

In `src/App.tsx`:

```tsx
import { ErrorBoundary } from './components/ErrorBoundary';

// In the render method:
return (
  <ErrorBoundary
    onError={(error, info) => {
      // Log to console / could send to error reporting service
      console.error('[App] Uncaught error:', error, info);
    }}
  >
    <div className={cn('min-h-screen flex flex-col', themeClass)}>
      {/* rest of app */}
    </div>
  </ErrorBoundary>
);
```

### Step 3: Add Global Error Handlers

In `src/main.tsx` or create `src/lib/globalErrors.ts`:

```typescript
// src/lib/globalErrors.ts

export function setupGlobalErrorHandlers() {
  // Catch uncaught JavaScript errors
  window.addEventListener('error', (event) => {
    console.error('[Global] Uncaught error:', event.error);
    // Could show a non-blocking toast here for unexpected errors
  });

  // Catch unhandled promise rejections
  window.addEventListener('unhandledrejection', (event) => {
    console.error('[Global] Unhandled promise rejection:', event.reason);
    // Prevent default browser handling (which shows a console error)
    event.preventDefault();
  });
}

// Call this in main.tsx after app mounts
```

### Step 4: Add Granular Error Boundaries for Major Sections

Consider adding error boundaries around major sections that are more likely to fail independently:

```tsx
// In App.tsx:
<ErrorBoundary fallback={<BoardErrorFallback />}>
  {view === 'board' && <Board />}
</ErrorBoundary>

<ErrorBoundary fallback={<DrawerErrorFallback />}>
  {drawerTaskId && <Drawer />}
</ErrorBoundary>
```

This way, if the board crashes, the drawer (with unsaved edits) stays intact.
