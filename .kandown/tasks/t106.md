# Context

When `reloadBoard` fails (e.g., permission revoked, file read errors, corrupted data), the board becomes **completely empty** — no tasks, no columns. The previous board state is gone with no recovery path. The user sees an empty kanban board and must figure out what happened.

**Current code** (`src/lib/store.ts:363-392`):
```typescript
reloadBoard: async () => {
  const { tasksDirHandle, config } = get();
  if (!tasksDirHandle) return;
  try {
    const tasks = await readAllTasks(tasksDirHandle);
    // ... build columns
    set({ boardTitle: 'Project Kanban', columns });
  } catch (e) {
    get().toast('Failed to load board: ' + (e as Error).message, 'error');
    // Board is now EMPTY — columns = whatever was set before this action
    // If this is the first load, columns = []
    // No retry, no partial load, no indication of what went wrong
  }
}
```

**Problems:**
1. **Previous state is destroyed** — if the user had tasks visible and reload fails, they see an empty board
2. **No retry logic** — transient errors (temporary permission loss, disk busy) are not retried
3. **No distinction between "empty board" and "failed load"** — both show zero tasks
4. **`readAllTasks` uses `Promise.all`** — if ANY task read fails (not just "file not found"), the entire operation rejects
5. **No loading state** — user can't tell if reload is in progress or if board is genuinely empty

## Subtasks

- [ ] Add `isReloading` state to store for loading indicator
- [ ] Preserve previous columns on reload failure instead of clearing
- [ ] Add `lastReloadError` state to track reload failure reason
- [ ] Implement automatic retry with exponential backoff (max 3 attempts)
- [ ] Update `readAllTasks` to use `Promise.allSettled` instead of `Promise.all`
- [ ] Show partial board on partial success (some tasks loaded, some failed)
- [ ] Add "Reload Failed" indicator in Header with retry button
- [ ] Add "board empty vs board failed" distinction in UI

## Why This Needs Fixing

**Severity: High**

When the board fails to load, the user sees an empty kanban with no tasks. They don't know if:
- The project genuinely has no tasks
- The board failed to load
- Permissions were revoked
- The disk is corrupted

A reload failure should **preserve the previously loaded board** and show a clear error indicator. The user should be able to retry without losing their current view.

## How to Fix

### Step 2: Preserve State on Failure

```typescript
reloadBoard: async () => {
  const { tasksDirHandle, config, columns } = get();
  if (!tasksDirHandle) return;

  // Mark as loading
  set({ isReloading: true, lastReloadError: null });

  try {
    const taskIds = await listTaskIds(tasksDirHandle);
    
    // Use allSettled to tolerate individual task failures
    const settled = await Promise.allSettled(
      taskIds.map(id => readTaskFile(tasksDirHandle, id))
    );

    const tasks: ParsedTask[] = [];
    const failedIds: string[] = [];

    settled.forEach((result, i) => {
      if (result.status === 'fulfilled' && result.value) {
        tasks.push(result.value);
      } else {
        failedIds.push(taskIds[i]);
      }
    });

    const parsedTasks = tasks.map(taskToBoardTask);
    const newColumns = buildColumnsFromTasks(parsedTasks, config.board.columns);
    
    set({ 
      boardTitle: 'Project Kanban', 
      columns: newColumns, 
      isReloading: false,
      lastReloadError: failedIds.length > 0 
        ? `${failedIds.length} task(s) could not be loaded` 
        : null,
    });

    // Content loading...
    const nextContents = new Map(taskContents);
    // ... load contents for available tasks

  } catch (e) {
    // PRESERVE previous columns — do NOT set columns to []
    set({ 
      isReloading: false,
      lastReloadError: `Failed to reload: ${(e as Error).message}`,
      reloadRetryCount: get().reloadRetryCount + 1,
    });
    get().toast('Board reload failed — showing last loaded state', 'warning');
  }
}
```

### Step 3: Add Retry Logic

```typescript
reloadBoardWithRetry: async (maxRetries = 3) => {
  const { reloadRetryCount } = get();
  
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    await get().reloadBoard();
    
    if (!get().lastReloadError) return; // Success
    
    if (attempt < maxRetries - 1) {
      const delay = 1000 * Math.pow(2, attempt); // 1s, 2s, 4s
      await new Promise(r => setTimeout(r, delay));
    }
  }
  
  // All retries exhausted
  get().toast(`Board reload failed after ${maxRetries} attempts`, 'error', 10000);
}
```
