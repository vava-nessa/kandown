# Context

At the bottom of `src/lib/store.ts` (~line 930-932), there's a module-level `listRecentProjects()` call that hydrates recent projects on store initialization:

```typescript
listRecentProjects().then(items => {
  useStore.setState({ recentProjects: items });
});
```

This has **no `.catch()` handler**. If `listRecentProjects()` rejects (IndexedDB blocked, disabled, storage quota exceeded, private browsing mode), the promise rejection is unhandled and can crash the store initialization.

Additionally, IndexedDB operations in `src/lib/filesystem.ts` have no protection against:
- **Private browsing mode** — IndexedDB may be unavailable or limited
- **Storage quota exceeded** — transactions fail silently
- **Browser blocking** — some browser settings block IndexedDB
- **Database version conflicts** — upgrade failures

**Current behavior:**
1. App loads
2. Module-level `listRecentProjects()` fires
3. IndexedDB is blocked (e.g., private browsing)
4. Promise rejects with no handler
5. Browser logs "Unhandled Promise Rejection" warning
6. Store initializes with `recentProjects = []` (default)
7. Functionally OK but console pollution and potential for hard crash in strict environments

## Why This Needs Fixing

**Severity: High**

While the current behavior is mostly "silent failure with console warnings," unhandled rejections are a code quality issue and can cause hard crashes in strict environments or future browser updates that may make unhandled rejections throw by default. Recent projects is a convenience feature — it should never prevent the app from starting.

## How to Fix

### Step 1: Add .catch() to Module-Level Hydration

```typescript
// At bottom of store.ts
listRecentProjects()
  .then(items => {
    useStore.setState({ recentProjects: items });
  })
  .catch(e => {
    console.warn('[Store] Failed to load recent projects:', e);
    // Graceful degradation — recent projects simply unavailable
    useStore.setState({ recentProjects: [], idbUnavailable: true });
  });
```

### Step 2: Add IndexedDB Availability Check

```typescript
// In filesystem.ts
let idbAvailable: boolean | null = null;

function isIndexedDBAvailable(): boolean {
  if (idbAvailable !== null) return idbAvailable;
  try {
    if (typeof indexedDB === 'undefined') {
      idbAvailable = false;
      return false;
    }
    // Try opening a test database
    const testReq = indexedDB.open('__kandown_test__', 1);
    testReq.onerror = () => { idbAvailable = false; };
    testReq.onsuccess = () => { idbAvailable = true; testReq.result.close(); };
    return true; // Optimistic — will be corrected on first actual use
  } catch {
    idbAvailable = false;
    return false;
  }
}

function openDB(): Promise<IDBDatabase> {
  if (!isIndexedDBAvailable()) {
    return Promise.reject(new Error('IndexedDB is not available'));
  }
  // ... existing code
}
```

### Step 3: Wrap All saveRecentProject Calls

In `store.ts openFolder`:
```typescript
try {
  await saveRecentProject({ handle: projectHandle, name: projectName, lastOpened: Date.now() });
} catch (e) {
  console.warn('[Store] Failed to save recent project:', e);
  // Don't block folder opening — just skip saving to recent
}
```

In `store.ts openRecentProject`:
```typescript
try {
  await saveRecentProject({ handle: project.handle, name: projectName, lastOpened: Date.now() });

  console.warn('[Store] Failed to update recent project:', e);

```
