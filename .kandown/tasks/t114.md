# Context

The Ink-based TUI screens (`src/cli/screens/`) have minimal error handling. If any screen component throws during render or in an input handler, the TUI crashes to the terminal with a stack trace. There's no recovery — the user must restart the TUI.

**Affected screens:**

### board.tsx — No error boundary around render
If `readBoard()` returns unexpected data or a task has malformed frontmatter, the render function can throw. Ink doesn't have built-in error boundaries like React.

### board.tsx:353-434 — useInput handlers with no try/catch
```typescript
useInput((input, key) => {
  // Multiple handlers that call functions which can throw:
  // - readBoard() on reload
  // - launchAgent() on agent select
  // - readTask() on task detail
  // None are wrapped in try/catch
});
```

### settings.tsx:166-173 — persistConfig with no try/catch
```typescript
const persistConfig = () => {
  saveConfig(kandownDir, localConfig); // writeFileSync — can throw on disk full
};
```
If this throws, the TUI crashes.

### agent-picker.tsx:37-61 — useInput callbacks
```typescript
useInput((input, key) => {
  if (key.return && selectedIndex >= 0) {
    onSelect(agents[selectedIndex].id); // Can throw if launchAgent fails
  }
});
```

### app.tsx — No error handling in screen routing
```typescript
if (screen === 'board') return <BoardScreen kandownDir={kandownDir} />;
if (screen === 'settings') return <SettingsScreen kandownDir={kandownDir} />;
return <BoardScreen kandownDir={kandownDir} />;
```

## Why This Needs Fixing

**Severity: Low**

TUI crashes are annoying but not data-destroying. The user just needs to restart `kandown board`. However, a robust TUI should degrade gracefully — show an error message in the terminal UI and offer recovery (reload, go back) instead of dumping a stack trace.

## How to Fix

### Step 1: Add Error State to Board Screen

```typescript
export default function BoardScreen({ kandownDir }: Props) {
  const [error, setError] = useState<string | null>(null);
  const [columns, setColumns] = useState<BoardColumn[]>([]);
  // ...

  const loadBoard = useCallback(() => {
    try {
      const board = readBoard(kandownDir);
      setColumns(board.columns);
      setTasks(board.tasks);
      setError(null);
    } catch (e) {
      setError(`Failed to load board: ${e.message}`);
    }
  }, [kandownDir]);

  useEffect(() => { loadBoard(); }, [loadBoard]);

  // Error state render
  if (error) {
    return (
      <Box flexDirection="column" padding={1}>
        <Text color="red">Error: {error}</Text>
        <Text dimColor>Press 'r' to retry or 'q' to quit</Text>
      </Box>
    );
  }

  // Normal render
  // ...
}
```

### Step 2: Wrap useInput Handlers

```typescript
useInput((input, key) => {
  try {
    if (key.leftArrow || input === 'h') { /* ... */ }
    if (key.rightArrow || input === 'l') { /* ... */ }
    if (key.upArrow || input === 'j') { /* ... */ }
    if (key.downArrow || input === 'k') { /* ... */ }
    if (input === 'r') { loadBoard(); }
    // ... etc
  } catch (e) {
    setStatusMsg(`Error: ${e instanceof Error ? e.message : String(e)}`);
    setTimeout(() => setStatusMsg(''), 4000);
  }
});
```

### Step 3: Wrap Settings persistConfig

```typescript
const persistConfig = () => {
  try {
    saveConfig(kandownDir, localConfig);
  } catch (e) {
    setSaveError(`Failed to save: ${e.message}`);
    setTimeout(() => setSaveError(null), 4000);
  }
};
```
