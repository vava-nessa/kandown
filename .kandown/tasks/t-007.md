# Plan CLI kanban board display and interactions

## Context

Plan the next Kandown CLI TUI screen so users can view and interact with the markdown kanban board directly from the terminal.

The CLI already has an Ink-based settings flow and an app-level screen switch, so the board experience should reuse the existing TUI structure instead of introducing a separate UI framework.

## Subtasks !!!

- [x] Define the board screen architecture and how it plugs into the existing CLI app router
- [x] Support two layout modes: kanban columns and focused single-column view
- [x] Add a keyboard shortcut to switch between layout modes
- [x] Load and parse `.kandown/board.md` from the active project folder
- [ ] Render columns and task cards with stable terminal-friendly truncation
- [ ] Add keyboard navigation between columns and tasks
- [ ] Open a task detail panel from the selected card
- [ ] Move tasks between columns and persist changes back to `board.md`
- [ ] Add quick task creation from the CLI board screen
- [ ] Add search and filtering, using Fuse.js if it stays the best fit after implementation review

## Notes

Keep the first implementation practical: one TUI board screen inside the existing CLI tool, backed by the current markdown parser and serializer where possible.

Interaction defaults to keyboard-first:

- Arrow keys navigate columns and cards.
- Enter opens or closes task details.
- A dedicated shortcut switches between all-columns view and focused-column view.
- Move/create actions must write valid Kandown markdown and avoid corrupting task files.

Before coding, confirm the exact shortcut map and persistence behavior so the first version is useful without being surprising.
